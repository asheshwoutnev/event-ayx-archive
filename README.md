# event-ayx-archive

> 一个用于归档和发布独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于收集、整理和发布独立形式的 HTML 页面。这些页面多为一次性活动页面、静态展示页面或特定场景下的前端成果快照。

仓库本身不针对任何特定域名或网站，仅作为页面文件的存档与分发点。

## 目录说明

```
event-ayx-archive/
├── pages/          # 存放归档的 HTML 页面文件（按文件夹组织）
├── assets/         # 页面依赖的静态资源（CSS、JS、图片等）
├── index.html      # 页面索引或入口文件（如有）
└── README.md       # 本文件
```

- **pages/**：每个子文件夹代表一个独立归档页面，文件夹名一般为页面主题或日期标识。
- **assets/**：存放多个页面可能共用的资源文件，若页面自带资源则置于其对应文件夹内。

## 页面归档说明

- 所有页面均为独立可运行的 HTML 文件，打开即可在浏览器中查看。
- 页面内容可能来自历史活动、临时项目或实验性尝试，不保证长期可用性或兼容性。
- 归档时尽量保留原始结构和资源，不做额外修改或优化。

### 如何浏览

1. 克隆或下载本仓库。
2. 在 `pages/` 目录下找到感兴趣的文件夹。
3. 直接打开其中的 `index.html` 或主 HTML 文件。

## 维护说明

- 本仓库不定期更新，新增页面会以新的文件夹形式加入 `pages/`。
- 不对已归档页面进行主动修改或删除，除非原始文件存在严重错误或安全风险。
- 欢迎提交 Issue 或 Pull Request 来补充、修正或建议归档内容。

## 注意事项

- 归档页面中的链接、资源引用可能已失效，建议在本地查看。
- 本仓库仅为技术存档用途，不构成对页面内容的任何背书或推荐。
